Форма Справочника.
Уникальность поля обычно обеспечивается базой данных (это назначение полю свойства Уникальное, Ключевое и т.д. в зависимости от используемой базы). Если такой возможности нет, как правило делается запрос на поиск максимального значения числового поля и к нему прибавляется 1. Но это слишком просто ) Вы просите сделать красиво).
Иногда Заказчик требует неукоснительного выполнения всех пунктов ТЗ. Чтобы сохранить в таблице Задания указанную очередность платформ, я решил дать возможность Пользователю самостоятельно вводить ЧисловойКод в Таблицу. Если введенный код уже есть в Справочнике, ищется новое значение от минимального в таблице до минимальной допустимой границы. Если в этом интервале не найдется свободный номер, поиск повторяется уже от 2-го минимального значения до 1-го минимального значения в Справочнике. И так далее. Найденный номер выводится в поле ЧисловойКод.
Предусмотрена возможность ускоренного ввода данных справочника. При изменении полей Ключевой или ЧисловойКод и заполненном поле Наименование происходит запись в Справочник. То же при изменении поля Наименование. Быстрое перемещение по полям формы клавишей Enter происходит в логической последовательности: Ключевой, ЧисловойКод, Наименование.
 Внешняя обработка.
По заданию разрешена только 1 кнопка. Поэтому программа сначала по запросу собирает данные из справочника. Затем помещает в Константу.ПлатформыРезерв ссылки на элементы справочника в случайным образом выбранном одном из форматов (Строка с GUIDами элементов, разделённые символом «;»; Массив ссылок; Таблица значений).
Затем из этой Константы извлекается ее значение и в зависимости от типа данных строится HTML документ, который выводит на форму обработки необходимую информацию. 
В ТЗ требуется выводить часть информации по возрастанию, а часть по убыванию поля ЧисловойКод. Обычно для этого выполняют 2 отдельных запроса. С целью уменьшения количества вызовов сервера и ускорения работы, в программе выполняется только 1 запрос.
